# React + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react/README.md) uses [Babel](https://babeljs.io/) for Fast Refresh
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react-swc) uses [SWC](https://swc.rs/) for Fast Refresh


무료 render 서버 사용으로 첫 요청 시 50초~1분 대기가 발생할 수 있습니다. 혹시 처음 시도시에 작동 되지않는다면 한번더 시도해주세요.

35~50대 여성 반려견 보호자들이 겪는 영양제 선택의 어려움을 AI Agent로 해결하는 맞춤 추천 서비스입니다.
반려견마다 필요한 영양 성분이 다름에도 불구하고, 정보 과잉과 전문 지식 부족으로 인해 보호자들은 확신 없이 구매하는 경우가 많습니다. PawCare AI는 반려견의 프로필(견종, 나이, 체중, 건강 관심사)을 기반으로 GPT-4o가 맞춤 영양 분석을 수행하고, 실제 쇼핑몰 데이터를 크롤링하여 최적의 제품을 추천합니다.

기술 스택
구분기술FrontendReact (Vite), Lucide ReactBackendNode.js, ExpressAIOpenAI GPT-4o API크롤링Cheerio, Axios외부 API네이버 쇼핑 검색 API호스팅Vercel (FE), Render (BE)

핵심 기능
1. 반려견 프로필 기반 맞춤 추천

견종, 나이, 체중, 건강 관심사 입력
견종 직접 타이핑 및 자동완성 지원 (목록에 없는 견종도 입력 가능)
건강 관심사 다중 선택 및 직접 입력 지원

2. 3개 플랫폼 실시간 크롤링 연동

네이버 쇼핑 (공식 API): 가장 대중적인 쇼핑 플랫폼으로 다양한 제품과 가격 비교 제공
이퀄펫: 한국에서 가장 많이 사용되는 반려동물 전문 영양제 브랜드로, 견종별 맞춤 영양제 특화
리브펫: 국내 최다 리뷰(2만개, 평점 4.8)를 보유한 반려동물 영양제 전문몰로 신뢰도 높은 데이터 제공

3. AI 맞춤 영양 분석

GPT-4o가 반려견 정보 + 실시간 크롤링 데이터를 분석
제품 추천 이유, 기대 효과를 소비자 언어로 설명
네이버쇼핑 vs 이퀄펫/리브펫 제품 비교 분석 제공
주의사항 및 리스크 안내

4. 견종별 건강 취약 부위 시각화

견종 선택 시 해당 견종의 건강 취약 부위를 SVG 도식으로 시각화
각 부위별 권장 영양 성분 안내
보호자가 직관적으로 이해할 수 있도록 설계

5. 사용자 리뷰 기반 AI 재추천 시스템

실제 구매 후 별점, 효과, 상세 후기 등록 가능
별점 4점 이상 제품에 "⭐ 사용자 별점 높은 제품" 뱃지 자동 표시
축적된 리뷰 데이터를 AI가 분석하여 추천 신뢰도 향상


설계 의도
왜 이 3개 쇼핑몰을 선택했나?

네이버 쇼핑: 국내 최대 쇼핑 플랫폼으로 가장 광범위한 제품과 가격 비교가 가능하며, 공식 API를 제공하여 안정적인 데이터 수집이 가능
이퀄펫: 견종별 맞춤 영양제를 전문으로 하는 브랜드로, MVP 컨셉(맞춤 추천)과 가장 잘 부합
리브펫: 리뷰 2만개, 평점 4.8의 신뢰도 높은 영양제 전문몰로 성분 정보가 상세하여 AI 분석에 활용

단순 추천이 아닌 "결정을 도와주는 시스템"
추천 이유, 기대 효과, 주의사항까지 함께 제공하여 보호자가 확신을 가지고 구매할 수 있도록 설계했습니다.

개발 중 어려웠던 점
1. 네이버 쇼핑 크롤링 차단 문제
초기에는 Cheerio로 네이버 쇼핑 HTML을 직접 크롤링하려 했으나, 네이버가 봇을 감지하여 418 에러로 차단했습니다. 네이버 쇼핑은 JavaScript로 동적 렌더링되기 때문에 단순 HTML 파싱으로는 데이터를 가져올 수 없었습니다. 결국 네이버 개발자 센터에서 공식 쇼핑 검색 API를 발급받아 안정적으로 연동했습니다.
2. 구매하기 링크 연동 문제
GPT가 추천 제품의 링크를 생성할 때 실제 URL이 아닌 "링크는 제공되지 않았습니다"와 같은 텍스트를 반환하는 문제가 발생했습니다. 이를 해결하기 위해 GPT 응답에서 링크 필드를 제거하고, 네이버 API가 반환하는 실제 상품 URL을 추천 결과와 직접 매핑하는 방식으로 변경했습니다. 또한 링크가 http로 시작하는지 검증하는 로직을 추가하여 잘못된 링크가 노출되지 않도록 처리했습니다.
3. WSL 네트워크 환경 이슈
로컬 개발 환경(WSL)에서 외부 API(OpenAI, 네이버 API) 접근이 차단되는 문제가 발생했습니다. 이로 인해 로컬 테스트가 불가능하여 Render에 배포된 환경에서 테스트를 진행했습니다.

강조하고 싶은 부분

크롤링 + AI의 유기적 결합: 단순히 API를 호출하는 것이 아니라, 실시간으로 크롤링한 실제 시장 데이터를 AI가 분석하여 개인화된 추천을 생성합니다.
사용자 리뷰 → AI 재추천 루프: 사용자가 남긴 리뷰 데이터가 다음 사용자의 추천 결과에 반영되는 선순환 구조를 구현했습니다.
신뢰도 중심 설계: 추천 이유, 성분 분석, 견종별 취약 부위 시각화, 사용자 별점 뱃지 등 다각도로 신뢰도를 높이는 장치를 마련했습니다.


개선하고 싶은 부분
1. 해외 직구 사이트 연동
일부 견종(비숑, 시츄 등)의 경우 국내 전용 제품이 부족합니다. iHerb, Chewy 등 해외 직구 사이트를 추가로 연동하여 사용자의 선택 폭을 넓히고 싶습니다.
2. 과학적 근거 제공
각 영양 성분이 반려견에게 왜 좋은지에 대한 신뢰도를 높이기 위해, 관련 수의학 논문이나 공인된 뉴스레터를 제품 설명 옆에 각주 형태로 제공하면 좋을 것 같습니다.
3. 로그인 시스템 도입
로그인 시스템을 도입하면 사용자의 좋아요 품목 및 구매 이력을 학습하여, 개인 선호도에 맞는 제품을 더 정교하게 추천할 수 있습니다. 협업 필터링(Collaborative Filtering) 방식으로 유사한 반려견을 키우는 보호자들의 선택도 반영할 수 있습니다.
4. 리뷰 유도를 위한 쿠폰 시스템
실제 상용화 시 사용자 리뷰 데이터를 더 많이 확보하기 위해, 리뷰 작성 시 할인 쿠폰을 발급하는 시스템을 도입하면 데이터 품질과 사용자 참여도를 동시에 높일 수 있습니다.
5. 이퀄펫 가격 정보 개선
현재 이퀄펫 크롤링 시 가격 정보가 HTML 구조상 description 필드에 혼재되어 정확한 파싱이 어렵습니다. 정규표현식으로 일부 처리하고 있으나, Puppeteer 등 헤드리스 브라우저를 활용한 더 정확한 크롤링이 필요합니다.
